# Contributing to Bangla Summarizer

Thank you for your interest in contributing to **Bangla Summarizer**!
We welcome contributions of all kinds—whether it’s code, bug reports, feature suggestions, documentation, or translations.

## How to Contribute

### 1. Fork the Repository
Click the **Fork** button at the top right of this repository to create your own copy.

### 2. Clone the Fork
```sh
git clone https://github.com/<your-username>/bangla_summarizer.git
cd bangla_summarizer
```

### 3. Create a Branch
Use a descriptive branch name for your change.
```sh
git checkout -b fix-typo-in-readme
```

### 4. Make Your Changes
Edit code, documentation, or other files as needed.

### 5. Test Your Changes
Make sure all tests pass and your changes do not break existing functionality.

### 6. Commit and Push
Write clear commit messages in English or Bangla.
```sh
git add .
git commit -m "Fix typo in README"
git push origin <your-branch>
```

### 7. Submit a Pull Request
Open a pull request from your branch to the `main` branch of this repository. Please describe your changes clearly.

---

## Code Style

- Please follow Python's [PEP8](https://pep8.org/) style guide.
- Use clear, descriptive variable and function names.
- Write docstrings for new functions/classes.
- Write comments in English or Bangla.

## Issues & Feature Requests

- Check for existing issues before creating a new one.
- When reporting a bug, include:
  - Steps to reproduce
  - Expected and actual behavior
  - Environment details (Python version, OS, etc.)
- For feature requests, explain the use case and possible implementation ideas.

## Tests

- Add tests for new features and bug fixes.
- Make sure tests cover edge cases.
- Run all tests before pushing your code.

## Translations

If you want to help with Bangla translations or localization, please contribute to the `locales` directory.

---

## Community

- Be respectful and constructive in discussions.
- Use GitHub Discussions or Issues for questions and feedback.
